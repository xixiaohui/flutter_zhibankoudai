# 智伴口袋多语言（i18n）支持 — 设计文档

## 概述

为智伴口袋 App 添加完整的多语言支持，采用 Flutter 官方 `flutter_localizations` + ARB 方案。默认跟随系统语言，架构上支持扩展更多语言。

## 目标语言

共 12 种语言，中文为默认语言：

| 序号 | 语言 | Locale 代码 | RTL |
| --- | --- | --- | --- |
| 1 | 中文（默认） | zh | 否 |
| 2 | 英文 | en | 否 |
| 3 | 日语 | ja | 否 |
| 4 | 韩语 | ko | 否 |
| 5 | 西班牙语 | es | 否 |
| 6 | 法语 | fr | 否 |
| 7 | 德语 | de | 否 |
| 8 | 葡萄牙语 | pt | 否 |
| 9 | 俄语 | ru | 否 |
| 10 | 阿拉伯语 | ar | **是** |
| 11 | 印地语 | hi | 否 |
| 12 | 泰语 | th | 否 |

## 语言解析策略

- 自动读取系统 Locale，无需手动切换
- 匹配规则：`zh_*` → zh, `en_*` → en, `ja_*` → ja, `ko_*` → ko, `es_*` → es, `fr_*` → fr, `de_*` → de, `pt_*` → pt, `ru_*` → ru, `ar_*` → ar, `hi_*` → hi, `th_*` → th
- 未匹配 → 回退到中文（zh，默认语言）
- 阿拉伯语（ar）自动启用 RTL 布局（Material 框架原生支持）
- 不提供 UI 手动切换入口（遵循用户需求）

### 核心组件

| 组件 | 职责 |
|------|------|
| `l10n/*.arb` | 所有 UI 固定字符串的翻译（~200 条） |
| `AppLocalizations` | gen-l10n 自动生成的类型安全本地化类 |
| `modules_{lang}.json` | 59 个模块的名称、描述、生成提示词的按语言配置 |
| `LocaleProvider` | 暴露当前 locale 给服务层读取 |
| `main.dart` | 注册 `localizationsDelegates`、`supportedLocales` |

### 依赖变更

```yaml
# pubspec.yaml
dependencies:
  flutter_localizations:
    sdk: flutter
  intl: ^0.20.2  # 已存在，保持不变
```

启用 gen-l10n：
```yaml
flutter:
  generate: true
  assets:
    - assets/cloudData/modules/
```

在 `l10n.yaml` 中配置：
```yaml
arb-dir: lib/l10n
template-arb-file: app_zh.arb
output-localization-file: app_localizations.dart
nullable-getter: false
```

### 数据流

#### 语言解析
```
App 启动 → 读取系统 Locale
  → 匹配 supportedLocales
  → zh_CN → zh, en_US → en, ja_JP → zh（回退）
  → 写入 LocaleProvider（供服务层）
  → Widget 通过 AppLocalizations.of(context) 获取翻译
```

#### 模块配置加载
```
确定 locale → 拼接路径 'assets/cloudData/modules/modules_{locale}.json'
  → 加载 JSON → 解析为 ModuleConfig 列表
  → ModuleProvider 持有 → 页面通过 Provider 读取
```

#### AI 服务调用
```
用户触发 AI 生成 → 读取当前 locale（'zh' / 'en'）
  → 传给 CloudBase AI 接口
  → AI 按语言参数生成对应语言内容
```

### constants.dart 改造

`AppConstants.defaultModules` 从硬编码的 59 个模块配置中移除。改为：
- 保留 `moduleTypes` map 作为 schema 参考
- 实际模块数据由 `ModuleProvider` 在启动时从 `modules_{lang}.json` 加载
- `AppConstants.appName` 等运行时字符串迁移到 ARB

## ARB 文件设计

### 键名规范

- 小写驼峰（camelCase）：`generatePoster`、`noContent`
- 按功能域分组（通过注释分隔）
- 避免使用占位符拼接：用 ARB 的 `{param}` 语法

### ARB 内容覆盖范围

| 功能域 | 字符串数 | 示例 |
|--------|---------|------|
| 通用（取消/确定/重试等） | ~15 | cancel, confirm, retry |
| 导航栏 | ~6 | bottomNavHome, bottomNavDiscover |
| 首页 | ~20 | moreModules, noModule |
| 模块详情页 | ~10 | generatePoster, aiGenerate |
| 海报页 | ~12 | saveToAlbum, savedSuccessfully |
| 我的页面 | ~25 | themeSettings, clearCache |
| AI 朋友/职业 | ~20 | deleteChat, typeHint |
| XUI 页面 | ~60 | quickEntry, loadingMore |
| 主题 | ~3 | lightMode, darkMode, followSystem |
| 字段标签 | ~12 | author, singer, director |
| **总计** | **~183** | |

## 迁移计划（3 阶段）

### 阶段 1：基础设施搭建
- 添加 `flutter_localizations` 依赖
- 创建 `l10n.yaml`
- 启用 `flutter generate: true`
- 创建 12 个 ARB 文件（app_zh.arb 为 template，先放少量字符串验证）
- 修改 `main.dart`：添加 `localizationsDelegates`、`supportedLocales`（含所有 12 个 locale）
- 运行 `flutter gen-l10n` 验证生成成功
- 添加 `LocaleProvider`
- 验证阿拉伯语 RTL 布局切换正常

### 阶段 2：UI 字符串迁移
- 逐文件迁移所有硬编码字符串到 app_zh.arb（~183 条）
- 替换 `Text('中文')` → `Text(AppLocalizations.of(context)!.key)`
- 英文翻译写入 app_en.arb（人工校对）
- 其余 10 种语言先用机器翻译生成初版，标记为待校对
- 模块配置从 `constants.dart` 迁移到 12 个 JSON 文件
- 修改 `ModuleProvider` 按语言加载模块 JSON
- 修改底部导航栏标签
- 修改 AI 朋友/职业页面的提示词和对话模板

### 阶段 3：服务层适配 & 验证
- `ai_service.dart` 传递 locale 参数到 CloudBase AI
- `data_service.dart` 按语言加载模块 JSON
- 切换系统语言验证每种语言下所有页面显示正确
- 重点验证阿拉伯语 RTL 布局不出现 UI 错位
- 翻译校对（英文优先，其他语言按需）

### 不做的事
- 不提供 UI 语言切换入口
- 不翻译服务端返回的 AI 动态内容（已通过参数交由服务端处理）
- 不翻译 `lib/config/constants.dart` 中的 `version`、`env` 等非用户可见常量

## 测试策略

- 开发时通过 `flutter run` 切换系统语言验证
- 重点验证中/英/阿拉伯语（RTL）三种典型布局
- CI：`flutter gen-l10n` 后 `flutter analyze` 确保无编译错误
- ARB 文件通过 `@@locale` 字段保证语言标识正确

## 风险 & 注意事项

1. **ARB 热重载问题**：修改 ARB 后需要重新 `flutter gen-l10n`，不是热重载
2. **翻译工作量**：12 种语言 × ~183 条字符串 = ~2200 条翻译，建议英文人工校对，其余机器翻译 + 标记
3. **RTL 布局**：阿拉伯语需要验证所有页面的 RTL 布局，特别是自定义的 Poster 页面和 AI Chat 页面
4. **第三方组件不可控**：`flutter_localizations` 仅覆盖 Material 组件（按钮文字等），自定义 Widget 需自行翻译
5. **服务端配合**：AI 生成多语言内容需要 CloudBase AI 接口支持语言参数
6. **模块 JSON 文件名约定**：`modules_{locale}.json` 格式需和 `LocaleProvider` 保持一致
7. **字体兼容**：阿拉伯语、印地语、泰语需要确认当前字体支持，可能需引入 Noto Sans 系列字体
